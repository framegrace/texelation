Communication Model

How do apps add/remove actions to the pill?

A

ControlBus Messages

Apps send decorator.add / decorator.remove messages via the existing ControlBus. Texelation's pane receives them and manages the pill. Apps don't know about the pill — they just declare actions.

controlBus.Send("decorator.add", DecoratorAction{
  ID:    "search",
  Icon:  "󰍉",
  Help:  "Toggle search (F3)",
  OnClick: func() { ... },
})
B

PaneDecorator Interface

Apps implement a PaneDecorator interface. The pane queries the app for its decorator actions on each render. Simpler, no messaging.

type PaneDecorator interface {
  DecoratorActions() []DecoratorAction
}
C

Direct Pane API

Apps call pane.AddDecoratorAction() directly (via a setter interface). Most explicit, but couples apps to the pane concept.
